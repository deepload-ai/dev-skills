# Refactor2Docs - 验证清单

## 完整性
- [ ] 八层目录均存在；无法详写的层有缺口说明。
- [ ] 每个核心模块至少有 PRD、架构设计、详细设计或缺口说明、测试规格。
- [ ] 重构指南包含实施顺序、风险、回滚和测试门禁。
- [ ] 代码图谱包含模块依赖图、执行流程图、数据流图谱、符号索引和 `meta.json`。

## 证据完整性
- [ ] 每个架构结论有源码路径。
- [ ] 每个接口定义有源码路径；关键接口到行号。
- [ ] 每条重要依赖边有来源或搜索范围说明。
- [ ] 所有推测均标注“基于代码推测”。
- [ ] 无来源的结论已删除或列入待确认。

## 一致性
- [ ] Phase 1 模块数 = Phase 2 分析模块数，或差异有解释。
- [ ] 文档中的模块名、路径、接口名与源码一致。
- [ ] Mermaid 图中的节点与正文模块/符号一致。
- [ ] 测试用例覆盖重构高风险路径。

## 可用性
- [ ] Markdown 链接无断链。
- [ ] Mermaid 代码块语法基本可渲染。
- [ ] `meta.json` 可被 JSON parser 解析。
- [ ] 构建/测试命令来自源码配置或明确标注待确认。

## 最终报告格式
```markdown
## 完成
- 生成/更新：[路径]

## 核心发现
- [结论]（证据：`path:line`）

## 验证
- [检查项]：通过/失败，证据或说明

## 缺口/风险
- [缺口]：原因、补全条件

## 建议下一步
- [最小可执行动作]
```
